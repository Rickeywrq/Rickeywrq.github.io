java对象内存、引用传递与垃圾产生分析
发表于 2020-03-18 | 更新于 2020-03-19
java

内存空间：

堆内存：保存的是一个对象的具体信息，存在一个地址。在程序之中堆内存空间的开辟是通过new完成的

栈内存：保存的是一块堆内存的地址，即：通过地址找到堆内存，而后找到对象内容,可以简单的理解为：对象名称保存在了栈内存中

所有的对象在调用类中的属性或方法的时候必须要实例化完成后才可以执行

同一块堆内存空间可以被不同的栈内存所指向，也可以更换指向

1.类名 对象2=对象1

2.方法();//等价上面那句话

方法（类名 形参）{//定义方法

​ 形参.属性 //调用属性

}

方法执行完毕参数就断开，栈内存就不访问堆内存

一个栈内存只能对应一个堆内存

没有任何栈内存指向的堆内存，那个堆内存就是垃圾空间

垃圾产生越少越好

一个栈内存只能够保存有一个栈内存的地址数据，如果发生更改，则之前的地址数据将从此栈内存彻底消失
